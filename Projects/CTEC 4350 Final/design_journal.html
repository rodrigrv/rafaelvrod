Design Journal

DHTML effect(s)

Navigation

I used a couple of different DHTML effects in this project. The most prominent one is probably the navigation bar. When it is full screen, it applies a hover effect. I added a CSS transition as well to the hover effect. To me, I think this gives the user a clear understanding of what they are hovering over. This also fulfills the clients goal of wanting a simple, yet functional navigation bar with all the pages readily available.

Events Page

In the events page, the client asked for a page where they can readily view past, present, and future events. A calendar option came to mind at first but if the month was over, the user would have to click to find the month. I then decided to have links displaying as "Past, Present, Future, and Show All" so that when the user clicks these links, it shows what corresponding events are from that particular link.

Form Validation

In previous projects, I've done only server-side validation. With this project, the client asked that users fill out certain forms for items such as volunteering and donations. I used JavaScript to successfully validate any common errors that were present in all 6 forms. For example: when the user fills out the 'zip code' label, they need to match a regular expression for U.S. zip codes. If it doesn't match, it sends an error when the user tries to submit that information.

XML/AJAX Feature

One of the clients biggest concerns was how to display information to the users. One way I thought would help convey that is by using XML/AJAX on the volunteer page. In that page, the checkboxes asking what events you can help with displays additional information about that event when you click the checkbox. I believe this met the criteria for the client wanting to display information to the user and for this project in using this dynamic feature.

Database-supported Feature

When I thought about how to implement a database-supported feature, it seemed like a no-brainer since I had over 5 forms that had to process user generated content. In this project, I used a combination of PHP and MySQL to submit user information into a database. Once the information is sent to the database, I can pull the information and display it on the admin page.

Mock-up Feature

The admin page and the user profile page on my site are very similar. The client stressed that they at least wanted an admin account to see all the information. I couldn't get the user profile to fully work. So the mock-up page is what the user profile would look like for one user. It would show their previous donations (if any), and their volunteer information. Additionally, they would have the option to edit or cancel their volunteer information. This is where the mock-up feature comes into place. At the moment, I only have it displayed without it working.
